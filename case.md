- REPO STRUCTURE: This is "learn-claude-code" - a comprehensive learning guide for building AI agent harnesses. 20 progressive sessions (s01-s20) teach agent patterns, starting from basic agent loops to comprehensive systems with teams, memory, scheduling, and MCP plugins. NOT about training models, but building the operational infrastructure ("harness") that lets trained models operate in environments.
- LEARNING PROGRESSION: Sessions build incrementally: s01=agent_loop, s02=tool_use, s03=permissions, s04=hooks, s05=todo_write, s06=subagent, s07=skill_loading, s08=context_compact, s09=memory, s10=system_prompt, s11=error_recovery, s12=task_system, s13=background_tasks, s14=cron_scheduler, s15=agent_teams, s16=team_protocols, s17=autonomous_agents, s18=worktree_isolation, s19=mcp_plugin, s20=comprehensive (all mechanisms integrated). Each folder contains code.py (implementation), README docs in 3 languages (EN/ZH/JA), and practice.py exercises.

- CORE PHILOSOPHY: Repo explicitly rejects prompt-chaining frameworks ("Rube Goldberg machines with if-else branches"). Agency comes from model training (learned via gradients/RL), not orchestration code. Harness philosophy: infrastructure (tools, hooks, lifecycle, environment) + trained model = working agent. References DQN (2013), OpenAI Five (2019), AlphaStar (2019), Jueyu (2019), modern LLM agents (2024-25).

Base pattern: while stop_reason=="tool_use" loop for tool execution before stopping.
